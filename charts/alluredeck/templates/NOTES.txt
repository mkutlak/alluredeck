⚠  IMPORTANT — Secrets are auto-generated on first install.
   They are preserved on upgrade using Helm lookup, but are NOT portable
   across clusters or Helm releases. For production use, supply a pre-created
   Secret and reference it with api.security.existingSecret:

   kubectl create secret generic alluredeck-credentials \
     --namespace {{ .Release.Namespace }} \
     --from-literal=adminUser='admin' \
     --from-literal=adminPassword='<strong-password>' \
     --from-literal=viewerUser='viewer' \
     --from-literal=viewerPassword='<strong-password>' \
     --from-literal=jwtSecretKey='<64-char-random-string>' \
     --from-literal=jwtAccessTokenExpires='900' \
     --from-literal=jwtRefreshTokenExpires='2592000'

   Then set in your values file:
     api:
       security:
         existingSecret: alluredeck-credentials
{{- end }}
{{- else }}

NOTE: Using existingSecret={{ .Values.api.security.existingSecret | quote }}.

     api.secretRevision: "v2"   # increment on each rotation
   Or run: kubectl rollout restart {{ .Values.api.kind | default "Deployment" | lower }}/{{ include "alluredeck.api.fullname" . }} -n {{ .Release.Namespace }}
