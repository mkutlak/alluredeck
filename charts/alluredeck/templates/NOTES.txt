{{- if and (eq (.Values.api.kind | default "Deployment") "Deployment") (ne .Values.api.config.storageType "s3") }}

⚠  WARNING — Local storage with Deployment detected (api.kind=Deployment, storageType=local).
   The projects PVC uses ReadWriteOnce access mode, which can only bind to a single node.
   The chart automatically sets strategy.type=Recreate to avoid a multi-attach deadlock
   during rolling updates, but only ONE replica can run at a time.

   For zero-downtime deployments or high availability, switch to StatefulSet:
     api:
       kind: StatefulSet

   Or use S3/MinIO storage, which allows multiple replicas and enables autoscaling:
     api:
       config:
         storageType: s3
{{- end }}

⚠  IMPORTANT — Secrets are auto-generated on first install.
   They are preserved on upgrade using Helm lookup, but are NOT portable
   across clusters or Helm releases. For production use, supply a pre-created
   Secret and reference it with api.security.existingSecret:

   kubectl create secret generic alluredeck-credentials \
     --namespace {{ .Release.Namespace }} \
     --from-literal=adminUser='admin' \
     --from-literal=adminPassword='<strong-password>' \
     --from-literal=viewerUser='viewer' \
     --from-literal=viewerPassword='<strong-password>' \
     --from-literal=jwtSecretKey='<64-char-random-string>' \
     --from-literal=jwtAccessTokenExpires='900' \
     --from-literal=jwtRefreshTokenExpires='2592000'

   Then set in your values file:
     api:
       security:
         existingSecret: alluredeck-credentials
{{- end }}
